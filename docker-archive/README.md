# Docker Archive

Эта папка содержит старые Docker-файлы, которые больше не используются в проекте.

## Причина удаления Docker

Проект был переведен на нативный деплой (без Docker) для:
- Экономии RAM (~200-300MB)
- Упрощения архитектуры для VPS с одним приложением
- Ускорения запуска и отладки

## Содержимое архива

- `Dockerfile` - образ приложения
- `docker-compose.yml` - основная конфигурация
- `docker-compose.local.yml` - override для локальной разработки
- `docker-compose.prod.yml` - override для production
- `deploy.sh` - старый скрипт автоматического деплоя с Docker
- `deploy-to-server.sh` - вспомогательный скрипт деплоя
- `nginx.conf` - конфигурация Nginx для Docker Compose

## Новый подход к деплою

Вместо Docker теперь используется:
- **PM2** для управления процессом Node.js
- **Nginx** на хосте (не в контейнере)
- **PostgreSQL** нативно на сервере

Полная документация: [DEPLOYMENT_GUIDE_NO_DOCKER.md](../DEPLOYMENT_GUIDE_NO_DOCKER.md)

## Когда может понадобиться Docker

Эти файлы сохранены на случай, если в будущем понадобится:
- Развернуть несколько приложений на одном сервере
- Использовать микросервисную архитектуру
- Обеспечить строгую изоляцию окружений

В таком случае можно вернуть эти файлы в корень проекта и адаптировать под текущую версию.
